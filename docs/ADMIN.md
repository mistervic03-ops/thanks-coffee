# ADMIN.md

비개발자 관리자가 모카를 운영할 때 보는 문서다. Slack에서 무엇을 입력해야 하는지, 문제가 생기면 무엇을 확인해야 하는지 중심으로 정리한다.

## 1. 이 봇이 하는 일

모카(Mocha)는 Slack 안에서 직원들이 서로에게 감사와 칭찬을 보내고, 그 기록을 전사 공지 채널에 남기는 봇이다.

직원은 `/thanks` 명령어로 칭찬을 보낸다.

예:

```text
/thanks @홍길동 도와주셔서 감사합니다
/thanks @홍길동 3 빠르게 대응해주셔서 감사합니다
/thanks @홍길동 ☕☕☕ 좋은 리뷰 감사합니다
```

## 2. 관리자 명령어 한 눈에 보기

| 명령어 | 하는 일 |
|--------|---------|
| `/mocha` | 관리자 명령어 목록 보기 |
| `/mocha summary weekly` | 직전 주 요약을 전사 공지 채널에 게시 |
| `/mocha summary monthly` | 직전 월 요약을 전사 공지 채널에 게시 |
| `/mocha summary weekly preview` | 직전 주 요약 미리보기 (나만 보임) |
| `/mocha summary monthly preview` | 직전 월 요약 미리보기 (나만 보임) |
| `/mocha summary this-month preview` | 이번 달 요약 미리보기 (나만 보임) |
| `/mocha delete {id}` | 잘못 보낸 칭찬 삭제 |
| `/mocha pin` | 공지 채널에 봇 소개 메시지 게시 및 pin |

## 3. 자주 있는 상황별 대응

### 상황 1: 직원이 잘못 보낸 칭찬을 취소해달라고 요청했을 때

상황:

직원이 받는 사람, 수량, 메시지를 잘못 입력해서 이미 칭찬이 기록된 상태다.

확인할 것:

- 삭제하려면 recognition ID가 필요하다.
- Slack 공지 채널에 올라온 메시지에서는 recognition ID를 확인할 수 없다.
- recognition ID는 DB 조회로 확인해야 한다.
- 구체적인 DB 조회 방법은 `docs/OPERATIONS.md`의 삭제 운영 절차를 참고한다.

행동:

1. DB에서 삭제할 recognition ID를 확인한다.
2. Slack에서 아래 명령어를 실행한다.

```text
/mocha delete {id}
```

예:

```text
/mocha delete 42
```

### 상황 2: 봇이 응답하지 않을 때

상황:

직원이 `/thanks`를 입력해도 응답이 없거나, 관리자가 `/mocha`를 입력해도 반응이 없다.

확인할 것:

- 서버에서 봇 프로세스가 실행 중인지 확인한다.

```bash
ps aux | grep app.py
```

행동:

1. 프로세스가 없으면 봇이 꺼진 상태다. 아래 4번의 "봇 시작 / 재시작 방법"을 따라 다시 시작한다.
2. 프로세스가 있으면 봇은 실행 중이다. 로그를 확인한다.

```bash
tail -f mocha.log
```

### 상황 3: 칭찬이 공지 채널에 안 올라올 때

상황:

직원이 `/thanks`를 보냈고 기록은 된 것 같은데, 전사 공지 채널에 메시지가 보이지 않는다.

확인할 것:

- 모카 봇이 전사 공지 채널에 초대되어 있는지 확인한다.
- 로그에서 `feed_post_failed`가 있는지 확인한다.
- feed 게시에는 자동 재시도가 있다. 바로 실패로 판단하지 말고 잠시 기다려도 된다.
- 자동 재시도는 최대 3회, 10분 간격으로 실행된다.
- 3회 모두 실패하면 관리자에게 DM 알림이 온다.

행동:

1. 공지 채널에 모카 봇이 없으면 채널에 초대한다.
2. 로그에서 `feed_post_failed`를 확인한다.
3. 자동 재시도가 끝날 때까지 기다린다.
4. 3회 실패 DM을 받으면 아래 상황 4를 따른다.

### 상황 4: 관리자 DM 알림이 왔을 때

상황:

모카 봇이 관리자에게 DM으로 상태나 오류를 알렸다.

확인할 것:

- DM 문구를 확인한다.

행동:

| DM 알림 | 의미와 행동 |
|---------|-------------|
| `[mocha] 봇이 시작되었습니다.` | 정상 알림이다. 봇이 시작되었거나 재시작된 것이다. |
| `[mocha] DB 연결에 실패했습니다.` | DB 서버 상태 확인이 필요하다. DB가 켜져 있는지, 연결 주소가 맞는지 확인한다. |
| `[mocha] feed 게시가 3회 재시도 후 포기되었습니다.` | 해당 칭찬은 기록됐지만 공지 채널에는 올라가지 않은 상태다. 필요하면 관리자가 수동으로 공지 채널에 올려도 된다. |
| `[mocha] 처리되지 않은 예외가 발생했습니다.` | 개발자에게 로그를 전달해야 한다. `mocha.log`에서 같은 시간대의 내용을 함께 전달한다. |

## 4. 봇 시작 / 재시작 방법

서버에 접속한 뒤 아래 순서로 실행한다.

```bash
cd /home/bigxdata/mocha-recognition
source venv/bin/activate
python app.py
```

백그라운드로 실행하려면 아래 명령어를 사용한다.

```bash
nohup python app.py > mocha.log 2>&1 &
```

로그를 확인하려면 아래 명령어를 사용한다.

```bash
tail -f mocha.log
```

## 5. 처음 세팅할 때 (신규 관리자 인수인계용)

아래 순서로 진행한다.

### 1. `.env` 파일에 필수 환경변수 설정

| 환경변수 | 설명 |
|----------|------|
| `SLACK_BOT_TOKEN` | Slack 앱 봇 토큰 |
| `SLACK_APP_TOKEN` | Slack 앱 소켓 토큰 |
| `DATABASE_URL` | PostgreSQL 연결 주소 |
| `ANNOUNCEMENT_CHANNEL_ID` | 전사 공지 채널 ID |
| `ADMIN_USER_IDS` | 관리자 Slack ID. 여러 명이면 쉼표로 구분한다. 선택 항목이며, Slack workspace Admin/Owner는 자동으로 관리자 권한이 부여된다. |

### 2. 봇 시작

위 4번의 "봇 시작 / 재시작 방법"을 참고해 봇을 시작한다.

### 3. Slack에서 확인

Slack에서 아래처럼 테스트 메시지를 입력한다.

```text
/thanks @본인 테스트 메시지
```

전사 공지 채널에 메시지가 올라오는지 확인한다.

### 4. 공지 채널에 소개 메시지 pin

Slack에서 아래 명령어를 실행한다.

```text
/mocha pin
```

### 5. 관리자 권한 확인

Slack에서 아래 명령어를 실행한다.

```text
/mocha
```

관리자 명령어 목록이 보이면 정상이다.
